Lucy Haskew

WDD 131-A1

Overview

Purpose

The purpose of my site is to create a functioning photo portfolio where I can display, upload, and delete photos for my audience and clients to be able to view images.

Audience

My audience consists of current and potential clients to be able to view past or current photography work. This is only going to allow people to view images that I can upload or delete. I would want user authentication to allow certain users to download their own portfolios.

Dynamic elements

Some dynamic elements that would be using JavaScript would be image gallery sliders, review rotation, modal popups for viewing larger versions of photos, and interactive forms for uploading or deleting photos. Additionally, I would implement user authentication features for certain clients to log in and access their private galleries.

Wireframes

Create two wireframes for your site. One for each page and list them here

Home

The home page includes a welcome message with my name and a main portrait image that introduces my style. The dark section under the hero is for a short slogan that represents my brand. Below that, there's a review section that will rotate reviews. The next section highlights a favorite photo from a recent shoot, with a short description and button linking to the portfolio page. Footer includes copyright info and social media links.

Portfolio

The portfolio page is organized into a grid layout showing featured photos from each portfolio. Each image block includes a placeholder for the portfolio name. The page begins with an intro text and short paragraph explaining what the user is viewing. The images are uniform in size and aligned in two columns. Footer includes copyright and social media links.
